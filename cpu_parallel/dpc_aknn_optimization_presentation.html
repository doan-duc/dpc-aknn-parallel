Lý thuyết

Bước 1 — Version 1: On-The-Fly + Parallel

Version 1

Giải quyết: Tràn RAM

  • Bỏ D[n×n]: Tính khoảng cách on-the-fly, không lưu ma trận
  • Mỗi thread cấp mảng tạm Neighbor[n], tính dist → qsort → lấy top-k
  • RAM: 39.2 GB → vài MB

⚠ Vấn đề phát sinh

  • malloc(n×sizeof) gọi bên trong parallel for
  • T thread tranh nhau heap lock → serialize ẩn
  • qsort(n) cho mỗi điểm: O(n log n) × n = lãng phí

Bước 1 — Version 2: Pre-alloc Buffer

Version 2

✅ Khắc phục: Heap Lock Contention

  • Cấp phát bufs[T] — 1 buffer/thread TRƯỚC vùng parallel
  • Thread dùng bufs[omp_get_thread_num()]
  • Không còn tranh heap lock → throughput mở khóa

⚠ Vấn đề còn lại

  • Vẫn dùng qsort(n) = O(n log n) cho mỗi điểm
  • Vẫn gọi sqrt() cho mọi cặp khoảng cách (~4.9 tỷ lần)
  • Buffer = O(n) mỗi thread: Threads × n ×sizeof(Neighbor) MB

Bước 1 — Optimized: Max-Heap + Early Exit

Optimized

3 Tối ưu đồng thời

  • [FIX-4] Max-Heap O(n log k): Heap kích thước k, thay thế qsort O(n log n). k=15 → tiết kiệm 4.2× so sánh
  • [OPT-1] dist²: So sánh bình phương, chỉ sqrt() k lần cuối. Loại bỏ ~4.9 tỷ phép sqrt
  • [OPT-2] Early Exit: Cộng dồn từng chiều, vượt heap root → break.
Buffer: O(n)→O(k)/thread: 17.9 MB → 3.8 KB  |  T₁: 3647s → 2562s (−30%)
